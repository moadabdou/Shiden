    \vspace{0pt}
    {\fontsize{9.5}{11}\selectfont\color{shidenaccent}\textbf{01 // MEMORY MODEL}\par}
    \vspace{1.5mm}
    
    {\fontsize{19}{23}\selectfont\color{white}\textbf{Java 21 FFM API Foundation}\par}
    \vspace{2mm}
    \noindent\rule{\linewidth}{0.4pt}
    \vspace{2.5mm}
    
    \normalsize\color{shidentext}
    Standard JVM Garbage Collectors (GC) trace heap object graphs. As database size expands to millions of entries, tracing overhead causes Stop-The-World (STW) pauses that destroy tail latency (p99/p99.9).
    
    Shiden bypasses the JVM GC entirely using Java 21's Foreign Function \& Memory (FFM) API (\texttt{JEP 454}). Native memory is mapped off-heap using thread-confined arenas (\texttt{Arena.ofConfined()}).
    
    Following Shiden's \textbf{Partition Ownership Model}, each partition thread owns its own isolated arena, completely eliminating JVM thread-synchronization checks and lock contention.
    
    \takeaway{Thread-confined FFM Arenas eliminate STW GC pauses while guaranteeing zero lock contention across partition threads.}
\end{minipage}%

% PAGE 3: SLAB ALLOCATOR & HARDWARE ACCELERATION
% ==============================================================================
\noindent
\begin{minipage}[t]{0.48\textwidth}
    \vspace{0pt}
    {\fontsize{9.5}{11}\selectfont\color{shidenaccent}\textbf{02 // SLAB ALLOCATOR}\par}
    \vspace{1.5mm}
    
    {\fontsize{19}{23}\selectfont\color{white}\textbf{\$O(1)\$ Bitmapped Slab Allocator}\par}
    \vspace{2mm}
    \noindent\rule{\linewidth}{0.4pt}
    \vspace{2.5mm}
    
    \normalsize\color{shidentext}
    Dynamic off-heap allocation (\texttt{malloc}) causes severe external memory fragmentation over time.
    
    The \textbf{Slab Allocator} pre-allocates native memory and sub-divides it into fixed-size, cache-aligned slots. Slot availability is tracked using a primitive \texttt{long[]} bitmap with a \textbf{cursor-based search index} to guarantee true $O(1)$ allocations under high slab occupancy.
    
    Allocation uses the CPU hardware instruction \texttt{TZCNT} via Java's \texttt{Long.numberOfTrailingZeros()}, while byte offset math uses fast ALU bitwise shifts (\texttt{slotIndex << slotShift}).
    
    \takeaway{Cursor-based bitmapped slab allocation achieves $O(1)$ operations with zero internal fragmentation for fixed metadata.}
\end{minipage}%
\hfill
\begin{minipage}[t]{0.01\textwidth}
    \centering\color{shidenpurple!20}\rule{0.8pt}{2.8in}
\end{minipage}%
\hfill
\begin{minipage}[t]{0.48\textwidth}
    \vspace{0pt}
    {\fontsize{9.5}{11}\selectfont\color{shidenmuted}\textbf{\$O(1)\$ BITWISE ALLOCATION LOGIC}\par}
    \vspace{3.5mm}
    
    \begin{lstlisting}[language=Java]
// Hardware bit scan + cursor resumption
long freeBits = ~bitmaps[i]; // 1 = free
if (freeBits != 0) {
    int bitIndex = Long.numberOfTrailingZeros(freeBits);
    int slotIndex = (i << 6) + bitIndex;
    bitmaps[i] |= (1L << bitIndex);
    return (long) slotIndex << slotShift; // Fast shift
}
    \end{lstlisting}
